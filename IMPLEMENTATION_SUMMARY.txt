                              PROJECT OVERVIEW
═══════════════════════════════════════════════════════════════════════════════

OBJECTIVE:
Create a complete Admin Panel system for the NESTIQ Real Estate Portal that
allows system administrators to:
  ✓ Register with a unique admin license key
  ✓ Login separately from regular users
  ✓ Access a professional control dashboard
  ✓ Manage all system users (buyers and sellers)
  ✓ Manage all properties listed on platform
  ✓ View system-wide statistics

                          SECURITY FEATURES
═══════════════════════════════════════════════════════════════════════════════

✅ IMPLEMENTED SECURITY MEASURES:

1. ADMIN KEY VALIDATION
   • Unique key required for admin registration
   • Key validation performed server-side (RegisterServlet)
   • Invalid key returns error, prevents account creation
   • Key case-insensitive for usability

2. ROLE-BASED ACCESS CONTROL
   • Admin dashboard checks session role
   • Non-admin users cannot access /adminDashboard
   • Unauthorized access redirects to homepage
   • Session invalidation on logout

3. ADMIN ACCOUNT PROTECTION
   • Delete action disabled for admin users
   • Prevents accidental or malicious deletion of admins
   • Only non-admin users can be deleted

4. SESSION MANAGEMENT
   • Admin session created only after valid login
   • Session attributes: loggedUser, loggedRole, loggedEmail, loggedPassword
   • Session cleared on logout

5. INPUT VALIDATION
   • Email validation (format check)
   • Password requirements (8+ characters)
   • Admin key field required for admin registration
   • Form validation on both client and server side

6. FUTURE SECURITY ENHANCEMENTS (Recommendations):
   • Move admin key to environment variable
   • Implement audit logging for admin actions
   • Add IP whitelist for admin access
   • Implement session timeout
   • Add HTTPS requirement
   • Password hashing (SHA-256/bcrypt)
   • Multi-factor authentication

                      TROUBLESHOOTING GUIDE
═══════════════════════════════════════════════════════════════════════════════

ISSUE: Admin key field not appearing
SOLUTION:
  • Clear browser cache (Ctrl+Shift+Delete)
  • Ensure JavaScript is enabled
  • Check console for JavaScript errors
  • Try different browser

ISSUE: Can't login as admin
SOLUTION:
  • Verify admin account exists in users.txt
  • Check email spelling and capitalization
  • Verify password matches (or try admin key)
  • Check if admin role is set correctly in file

ISSUE: Admin dashboard shows no data
SOLUTION:
  • Check files exist: WEB-INF/users.txt, properties.txt
  • Verify file permissions allow reading
  • Check AdminDashboardServlet logs for errors
  • Restart Tomcat and retry

ISSUE: Delete buttons don't work
SOLUTION:
  • Check user is logged in as admin
  • Verify not trying to delete another admin
  • Check browser console for form submission errors
  • Verify DeleteUserServlet and deleteProperty servlet paths
